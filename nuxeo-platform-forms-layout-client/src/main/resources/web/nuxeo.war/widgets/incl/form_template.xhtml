Reusable template to display a form. Accepted parameters: - addForm: boolean stating if a form should be added (optional, defaults to false) - formId: the form id (required if addForm is true) - useAjaxForm: boolean stating if form should be ajaxified (optional, defaults to false) - eventsQueueId: id for events queue of the ajax form (optional, defaults to the form id suffixed by _eventsQueue) - reRender: ajax elements ids to rerender (optional) - formStyleClass: style class to put on the form (optional) - formEnctype: since 5.9.5, encoding type must be set to "multipart/form-data" for forms submissions containing multipart information. This is useful to prevent invalid ajax requests, see https://java.net/jira/browse/JAVASERVERFACES-2843 Templating zone to define: - form_template_content: inside the form if a form is defined
